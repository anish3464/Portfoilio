Carcinogen Detector – Product Safety Analyzer

AI-driven tool to identify carcinogenic ingredients using OCR + deep learning

🔧 Tech Stack

AI & ML

ChemBERTa LLaMA 3.2 HuggingFace

Backend

Python Flask Tesseract OCR

Data Sources

PubChem API SMILES IARC Data

🧪 Problem Solved

Many cosmetics, shampoos, and food items contain chemicals flagged as potential carcinogens. Most consumers are unaware of these risks.

This project allows users to scan a product label and get a detailed carcinogen risk report powered by LLMs and chemical datasets.

✨ Key Features

📷 OCR Ingredient Extraction

  • Extract ingredients from product label images
  • Clean and standardize chemical names
  • Handle multiple languages and formats

🧬 SMILES Mapping

  • Fetch chemical structures from PubChem
  • Generate molecular fingerprints
  • Resolve ambiguous chemical names

🤖 AI Classification

  • Fine-tuned ChemBERTa for carcinogen prediction
  • Training on IARC/NIH labeled datasets
  • Confidence scoring and risk levels

💬 LLM Explanations

  • LLaMA 3.2 explains why chemicals are flagged
  • Plain-language risk explanations
  • Source citations and confidence scores

💡 Technical Challenges

Name Standardization: Inconsistent ingredient names across regions/products
PubChem Resolution: Multiple matches for similar chemical names
Limited Training Data: Tuning ChemBERTa on sparse labeled datasets
LLM Grounding: Preventing hallucinations in explanations

📈 Results & Detection

90%+
Prediction accuracy on test set
Formaldehyde
Successfully detected in cosmetics
Parabens & SLS
Common dangerous compounds identified

📈 Future Potential

Deployment: Chrome extension for website scanning, mobile app integration

Network Intrusion Detection System (NIDS)

A machine learning-powered intrusion detection system to classify and analyze network activity

🔧 Tech Stack

Backend & ML

Python Flask scikit-learn

Data Processing

pandas joblib NSL-KDD Dataset

Deployment

Docker Heroku Flask Templates

🧭 Problem Context

This project addresses the growing need for real-time network defense due to increasing cyberattacks. Traditional rule-based IDSs are often limited in detecting zero-day threats.

My NIDS uses machine learning models trained on NSL-KDD, a standardized dataset used in academic research for anomaly-based intrusion detection.

📚 Dataset & Features

📊 NSL-KDD Dataset

  • Refined version of KDD Cup '99 dataset
  • Preprocessed network connection features
  • Protocol type, service, flag, byte counts
  • Login attempts and error rates

🏷️ Attack Classifications

  • Normal: Legitimate traffic
  • DoS: Denial of Service attacks
  • Probe: Surveillance and probing
  • R2L: Remote to Local attacks
  • U2R: User to Root attacks

✨ Key Features

🤖 Multiclass Classification

  • Detects 5 attack categories with high accuracy
  • Minimal false positives in production testing
  • Real-time prediction capabilities

🌐 Web Interface

  • User-friendly form for connection statistics
  • Instant prediction output display
  • Categorized input fields for easy use

📈 Explainable Features

  • Basic connection features (duration, bytes)
  • Content-based features (login attempts)
  • Time-based and host-based statistics

🐳 Dockerized Deployment

  • Complete containerization with Dockerfile
  • Heroku-ready with Procfile configuration
  • Scalable API for integration

💡 Technical Challenges

Dimensionality Reduction: Handling 41 feature vectors efficiently
Attack Generalization: Ensuring model works across different attack categories
UI Integration: Building usable interface on top of complex ML classifier
False Positive Control: Maintaining 99%+ accuracy while minimizing false alarms

📈 Results & Detection

99%+
Classification accuracy on test set
Satan & Portsweep
Successfully detected attack patterns
Neptune DoS
Identified denial of service attacks

Deployment-ready ML pipeline with Flask interface that can be integrated into real-world low-resource environments for traffic-level monitoring.

🌐 Project Purpose

While travel review sites like TripAdvisor focus on structured reviews, TravelTroppTalk is designed for open-ended Q&A style interaction — travelers helping each other in real time, like StackOverflow for tourism.

The platform enables authentic peer-to-peer knowledge sharing about public transport, local food, safety tips, and cultural insights across different countries.

✨ Key Features

👥 User Management

  • Secure user registration and authentication
  • Profile management with travel preferences
  • Session-based login system
  • Password hashing for security

💬 Discussion System

  • Post questions in categorized topics
  • Comment-based discussion threads
  • Real-time interaction between travelers
  • Upvoting and community moderation

🏷️ Organization Features

  • Country-based tagging and filtering
  • Category sorting (Food, Transport, Culture)
  • Advanced search functionality
  • Topic-based content discovery

🚀 Future Enhancements
